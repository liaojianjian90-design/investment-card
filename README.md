# 投资健康卡 5.4.2：仓位体检 + 总盈亏版

5.4.2 的定位已经从“自动买卖提醒系统”调整为：

```text
仓位统计 + 风险上限提示 + 操作前人工复核
```

网站只负责回答三个问题：

1. 现在仓位结构是什么样；
2. 哪些层级或标的接近上限；
3. 操作前需要注意哪些风险边界。

网站不再自动指挥加仓、减仓或止盈。真正操作前，先发最新 holdings.json 或 Bitget 截图，再人工复核。

---


## 5.4.2 新增：版块顺序与持仓排序优化

页面顺序调整为：

1. 账户总盈亏
2. 账户快照
3. 持仓与盈亏
4. 资产仓位结构
5. 投资健康评分
6. 仓位体检结论
7. 仓位风险提示
8. 操作前人工复核流程
9. 仓位体检规则
10. 仓位来源

“持仓与盈亏”默认按当前市值 / 仓位占比从高到低排序，最大仓位排在最上面，方便优先检查对账户影响最大的标的。

## 5.4.2 新增：账户总盈亏

页面顶部新增“账户总盈亏”模块，固定以：

```text
16000 USDT
```

作为整套账户的基准本金。

展示内容包括：

- 当前总资产
- 总盈亏金额 = 当前总资产 - 16000 USDT
- 总盈亏率 = 总盈亏金额 / 16000 USDT

这个模块只做结果统计，不触发任何买入、卖出、补仓或止盈建议。

如需调整基准本金，可修改：

```text
config/rules.json -> performanceBaseline.amount
```

## 5.4.2 核心变化

### 1. 只统计，不催买

保留：

- 总资产
- 现金比例
- BTC / ETH 核心层
- VOO / XAUT 稳定层
- AI 相关仓
- DOGE / BGB 投机清理层
- CRCL 加密金融基础设施卫星仓
- 单一标的占比
- 组合集中度
- 价格源和数据状态

关闭或弱化：

- 自动买入建议
- A/S 级急跌买入日志
- 执行纪律日志前端展示
- 频繁邮件交易提醒
- 前端手动交易录入
- Cloudflare Worker 同步

## 2. 操作前人工复核流程

每次想操作时，先按这个流程：

```text
第一步：先不下单
第二步：发最新 holdings.json 或 Bitget 截图
第三步：说明想操作哪个标的、方向和金额
第四步：先判断是否属于情绪交易
第五步：确认现金比例、单项上限、AI总仓、DOGE/BGB/RAM和CRCL上限
第六步：确认后再执行
第七步：成交后更新 config/holdings.json
```

---

## 3. 资产层级

当前系统显示六个层级：

1. 安全现金层
2. 核心增长层
3. 长期稳定层
4. AI相关主线仓
5. 投机清理层
6. 加密金融基础设施卫星仓

其中：

```text
DOGE / BGB = 投机清理层
CRCL = 加密金融基础设施卫星仓
```

CRCL 不与 DOGE/BGB/RAM 合并计算投机仓上限。

---

## 4. CRCL 独立卫星仓

CRCL 规则：

| 项目 | 数值 |
|---|---:|
| 目标上限 | 4% |
| 停止新增 | 4.5% |
| 硬上限 | 5% |

CRCL 是加密金融基础设施卫星仓，不是 AI 主攻仓，也不是 DOGE/BGB/RAM 投机仓。

---

## 5. GitHub Actions 频率

5.4.2 不再使用 5 分钟高频交易提醒。默认改为美股时段每 30 分钟刷新一次价格和风险状态：

```yaml
- cron: "17,47 13-22 * * 1-5"
```

这样网站仍能更新仓位体检数据，但不会变成交易指挥系统。
