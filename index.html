投资纪律系统 5.2

优先级：数据有效性 → 现金安全 → 回撤状态 → 投机仓限制 → 核心仓缺口 → 稳定层底仓 → 主题仓上限 → 趋势跟随。

一、硬性停止规则

  • 价格源失败：失败标的不触发买入邮件，只显示“等待下次刷新”。
  • 数据超过 30 分钟：禁止按信号买入。
  • 现金低于 35%：停止所有新增买入。
  • 现金低于 40%：暂停普通加仓，只允许观察。
  • 一天最多一笔主动交易：BTC 急跌或核心买点触发后，当天不再触发 AI 抽水机仓买入。

仓位来源

当前版本已关闭浏览器端交易录入、本地仓位编辑和第三方同步面板。网页、监控和邮件均以 GitHub 仓库中的 config/holdings.json 为唯一仓位来源。

更新方式：修改 config/holdings.json 后提交到 GitHub，再运行 GitHub Actions，系统会重新生成 data/snapshot.json、健康评分和邮件提醒。
