仓位体检规则 5.4

5.4 的职责很简单：只统计仓位、只提示风险边界、只提醒你操作前人工复核。它不是自动买卖系统，不再因为价格触发就告诉你应该买多少。

一、仓位体检边界

  • 现金层：40%-50% 为舒适区，35% 以下进入现金保护提醒。
  • 核心层：BTC/ETH 看合计仓位，合计达标后，单项偏低只显示“核心偏科”。
  • 稳定层：VOO/XAUT 只作为长期稳定仓，不抢 AI 主线资金。
  • AI相关仓：MU/DRAM/GLW/SMH 是第一主攻层，SNDK/WDC/MRVL/AVGO/ANET/TSM/ASML/SMH/SOXX/FN/AAOI/ASX 属于 AI 相关观察或第二梯队。
  • DOGE/BGB：只算投机清理层，不与 CRCL 合并。
  • CRCL：单独作为加密金融基础设施卫星仓，4.5% 停止新增，5% 硬上限。

四、操作前沟通原则

  • 先不下单：任何想加仓、减仓、止盈、抄底、追涨之前，先暂停。
  • 先发数据：发最新 holdings.json 或 Bitget 截图，至少包含数量、现价、成本、浮盈亏和现金。
  • 先判断性质：区分是真正仓位调整，还是亏损补仓、上涨追涨、情绪交易。
  • 先看上限：如果某个标的或层级已接近上限，默认不新增。
  • 执行后更新：真正成交后，再更新 config/holdings.json，让网站重新体检。

仓位来源

当前版本已关闭浏览器端交易录入、本地仓位编辑、第三方同步面板、自动执行日志和自动买卖建议。网页只读取 GitHub 仓库中的 config/holdings.json 做仓位体检。

更新方式：修改 config/holdings.json 后提交到 GitHub，再运行 GitHub Actions，系统会重新生成 data/snapshot.json 和健康评分。真正操作前，请先发最新仓位或截图进行人工复核。
